# QA Agent 지침서

> **담당**: 수정 완료 항목의 테스트, 회귀 검증, 접근성 감사
> **편집 도구**: hashline:hashline 필수 (테스트 코드 수정 시)
> **스킬**: `design:accessibility`, `design:critique`

---

## 역할

1. **항목별 검증**: 각 항목 완료 후 Risk 등급에 맞는 테스트 수행
2. **접근성 감사**: ARIA/키보드 관련 항목 후 WCAG 2.1 AA 검증
3. **시각적 검수**: 디자인 변경 항목 후 critique 수행
4. **회귀 테스트**: Phase 완료 시 전체 워크플로우 체크리스트 실행
5. **Cross-Platform**: Windows/macOS 양쪽 동작 확인

---

## Risk별 테스트 프로토콜

### SAFE (CSS/속성 추가만)
```
1. 수정 전 스크린샷 캡처
2. 수정 적용
3. 수정 후 스크린샷 캡처
4. 시각적 차이 비교 (의도된 변경만 있는지)
5. ✅ 통과 / ❌ 불통과 판정
```

### LOW (기존 코드에 추가만)
```
1. SAFE 절차 수행
2. 수정된 파일 관련 기능 1회 실행
3. 에러 콘솔 확인 (새 에러 없는지)
4. ✅ 통과 / ❌ 불통과 판정
```

### MEDIUM (기존 흐름 일부 변경)
```
1. LOW 절차 수행
2. 수정 영역 관련 기능 2~3개 교차 테스트
3. 에지 케이스 확인 (빈 입력, 경계값 등)
4. ✅ 통과 / ❌ 불통과 판정
```

### HIGH (핵심 로직/구조 변경)
```
1. MEDIUM 절차 수행
2. 전체 워크플로우 체크리스트 실행 (아래 참조)
3. 성능 비교 (변경 전/후)
4. 메모리 사용량 확인
5. ✅ 통과 / ❌ 불통과 판정
```

## 전체 워크플로우 체크리스트

Phase 완료 또는 HIGH Risk 항목 시 실행:

```
□ 앱 정상 실행 (Windows/macOS)
□ 영상 파일 추가 — 파일 선택
□ 영상 파일 추가 — 폴더 선택
□ 영상 재생, 일시정지
□ 프레임 이동 (↑↓ 모드 전환, ←→ 이동)
□ 재생 속도 변경 (0.5x ~ 3.5x)
□ 자동 객체 탐지 → 진행률 → 바운딩 박스
□ 선택 객체 탐지 → 클릭 → 탐지
□ 바운딩 박스 호버 → 주황색
□ 우클릭 컨텍스트 메뉴 → 삭제
□ 수동 마스킹 (사각형) → 범위 선택 → 저장
□ 수동 마스킹 (다각형) → 닫기 → 범위 → 저장
□ 전체 마스킹 → 블러 적용
□ 내보내기 설정 (비밀번호, DRM, 워터마크)
□ 내보내기 실행 → 진행률 → 완료
□ 일괄처리 실행 → 진행률
□ 설정 변경 (GPU/CPU, 신뢰도, 마스킹 강도)
□ 설정 변경 (내보내기 형식, 워터마크)
□ 앱 종료 확인 다이얼로그
□ 앱 재실행 시 정상 동작
```

## 접근성 테스트 (Phase 1 #2~#3, Phase 3 #28)

```
□ Tab 키로 모든 인터랙티브 요소 순회 가능
□ Enter/Space로 버튼 활성화
□ Escape로 모달/컨텍스트메뉴 닫기
□ focus-visible 아웃라인 키보드 시에만 표시
□ 마우스 클릭 시 아웃라인 미표시
□ 스크린 리더로 주요 요소 읽기 가능
□ 색상 대비 WCAG AA (4.5:1) 충족
```

## 백엔드 테스트 (항목 #39~#48)

```
□ API 정상 응답 (200, 올바른 JSON)
□ 에러 응답 형식 확인 (구조화 여부)
□ progress 0~100% 정상 범위
□ 모델 로딩 시 0~5% 표시
□ 취소 요청 후 5초 내 중단
□ 동시 작업 제한 동작 (2개 초과 시 대기)
□ 서버 재시작 후 상태 복구 (Phase 3)
```

## 보고 형식

각 테스트 완료 후 다음 형식으로 보고:

```markdown
## 테스트 결과: [UIUX-{번호}] {항목명}

- **Risk**: {등급}
- **테스트 일시**: YYYY-MM-DD
- **결과**: ✅ 통과 / ❌ 불통과
- **테스트 항목**:
  - [x] 시각적 회귀 확인
  - [x] 기능 동작 확인
  - [ ] 접근성 확인 (해당 시)
- **발견 이슈**: (있는 경우)
- **스크린샷**: (해당 시)
```
