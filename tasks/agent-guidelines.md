# SecuWatcher UI/UX 개선 — Agent Guidelines

> **프로젝트**: SecuWatcher Export (Electron 36 + Vue 3.5 + Vite 5 + FastAPI Python)
> **배포 대상**: Windows / macOS
> **국제화**: 기본 한국어(ko), 추가 영어(en) — 2개 언어만
> **최종 수정**: 2026-03-04

---

## 공통 규칙 (모든 에이전트 필수)

### 도구 사용
- **파일 편집**: 반드시 `hashline:hashline` 스킬을 사용할 것
- **파일 탐색**: hashline의 읽기 명령 또는 표준 Read/Grep 도구 사용
- **Task 관리**: `tasks/todo.md` 체크리스트 업데이트 필수
- **교훈 기록**: 실수/수정 발생 시 `tasks/lessons.md` 즉시 업데이트

### Workflow Orchestration

#### 1. Plan-First Mode
- 모든 비단순 작업(3단계 이상 또는 설계 판단 필요)은 계획 모드로 시작
- 문제가 발생하면 **즉시 중단**하고 재계획 — 강행 금지
- 빌드 단계뿐 아니라 검증 단계에서도 계획 모드 활용
- 모호함을 제거하기 위해 상세 명세를 사전에 작성

#### 2. Sub-Agent Strategy
- 메인 컨텍스트 윈도우를 깨끗하게 유지하기 위해 서브 에이전트에 자유롭게 위임
- 조사, 탐색, 병렬 분석 작업은 서브 에이전트로 이관
- 복잡한 문제는 서브 에이전트를 통해 추가 연산 자원 활용
- 서브 에이전트 1개당 1개 태스크에 집중

#### 3. Self-Improvement Loop
- 사용자로부터 수정을 받으면 **반드시**: `tasks/lessons.md`에 학습 패턴 기록
- 동일 실수가 반복되지 않도록 자기 규칙을 작성
- 오류율이 떨어질 때까지 교훈을 반복적으로 개선
- 세션 시작 시 관련 프로젝트의 교훈을 리뷰

#### 4. Pre-Completion Validation
- 작업이 동작함을 증명하지 않고는 절대 완료로 표시하지 않음
- 관련 있는 경우, main과 변경사항 간 동작 차이를 비교
- 스스로에게 질문: *"시니어 엔지니어가 이것을 승인할까?"*
- 테스트 실행, 로그 확인, 정확성 증명

#### 5. Pursue Elegance (Stay Balanced)
- 비단순 변경 시: 잠시 멈추고 *"더 우아한 접근법이 있을까?"* 자문
- 땜질처럼 느껴지면: *"내가 아는 모든 것을 고려해서 우아한 해법을 구현하자"*
- 단순하고 명백한 수정에는 이 과정을 생략 — 오버엔지니어링 방지
- 결과물을 제출하기 전에 비판적으로 자기 검토

#### 6. Autonomous Bug Fixing
- 버그 리포트를 받으면: 단계별 안내를 요청하지 말고 바로 수정
- 로그, 에러, 실패하는 테스트를 드러낸 후 해결
- 사용자에게 컨텍스트 전환을 요구하지 않음
- 지시 없이도 실패하는 CI 테스트를 선제적으로 발견하고 수정

## Task Management

1. **Plan First**: 작업 시작 전에 `tasks/todo.md`에 체크리스트 계획 작성
2. **Validate the Plan**: 구현 시작 전에 확인(오퍼레이터 승인) 받기
3. **Track Progress**: 진행하면서 완료 항목 체크오프
4. **Explain Changes**: 각 단계마다 고수준 요약 제공
5. **Document Results**: 완료 시 `tasks/todo.md`에 리뷰 섹션 추가
6. **Log Lessons**: 수정이 필요했던 경우 `tasks/lessons.md` 업데이트

## 커밋 규칙

```
[UIUX-{번호}] {설명} (Risk: {등급})

예: [UIUX-01] Vue 글로벌 에러 핸들러 설정 (Risk: LOW)
예: [UIUX-39] [BE] print→logging 전환 (Risk: SAFE)
```

---

## Risk별 테스트 범위

| Risk 등급 | 필수 테스트 | 방법 |
|-----------|------------|------|
| SAFE | 시각적 회귀 | 수정 전후 스크린샷 비교 |
| LOW | 해당 기능 수동 테스트 | 수정 파일 관련 기능 1회 실행 |
| MEDIUM | 관련 기능 전체 | 수정 영역 + 연관 기능 2~3개 교차 |
| HIGH | 전체 회귀 | 모든 주요 워크플로우 재확인 |

## 주요 워크플로우 체크리스트 (MEDIUM+ Risk 시 확인)

```
□ 영상 파일 추가 (파일 선택 / 폴더 선택)
□ 영상 재생, 일시정지, 프레임 이동
□ 재생 속도 변경 (0.5x ~ 3.5x)
□ 자동 객체 탐지 실행 → 진행률 → 바운딩 박스 표시
□ 선택 객체 탐지 실행
□ 바운딩 박스 호버 → 주황색
□ 우클릭 컨텍스트 메뉴 → 삭제
□ 수동 마스킹 (사각형/다각형) → 범위 선택 → 저장
□ 내보내기 (비밀번호, DRM, 워터마크)
□ 내보내기 실행 → 진행률 → 완료
□ 설정 변경 (GPU/CPU, 신뢰도, 마스킹 강도)
□ 앱 종료 확인 다이얼로그
```
