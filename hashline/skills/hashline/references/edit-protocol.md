# Hashline 편집 프로토콜 — 상세 레퍼런스

## djb2 해시 알고리즘

해시 계산 방법:
1. 줄 내용에서 모든 공백 제거
2. djb2 해시 계산: `h = 5381; for each char: h = (h * 33 + charCode) % 65536`
3. `h % 256`을 2자리 소문자 hex로 포맷

공백만 다른 변경(들여쓰기, 후행 공백)은 동일한 해시를 생성합니다.

## 앵커 검증

편집 명령이 `LINE:HASH`를 참조할 때:
1. `LINE`이 파일 범위 내인지 확인 (1 ~ 전체 줄 수)
2. 해당 줄 내용의 실제 해시 계산
3. 일치 → 앵커 유효
4. 불일치 → **Relocation** 시도

### Relocation

해시 불일치 시:
1. 전체 파일에서 예상 해시와 매칭되는 줄 스캔
2. 정확히 1개 발견 → 해당 줄로 재위치, stderr에 `RELOCATED N:HH -> M:HH` 보고
3. 0개 또는 2개 이상 → `anchor_mismatch` 오류, 5줄 컨텍스트 표시

## 배치 편집 의미론

### 전부 또는 없음 (All-or-Nothing)

배치의 모든 앵커는 편집 적용 **전에** 검증됩니다.
하나라도 검증 실패(Relocation 시도 후에도)하면 배치 전체가 거부되고 파일은 변경되지 않습니다.

### Bottom-Up 적용

검증 후 편집은 줄 번호 내림차순으로 정렬되어 적용됩니다.
이로써 삽입/삭제 시 이전 줄 번호의 안정성을 보장합니다.

### 오버랩 감지

적용 전, 모든 편집 범위의 오버랩을 검사합니다.
겹치는 줄 범위를 대상으로 하는 두 명령은 거부됩니다 (`insert_after` 제외 — 범위를 소비하지 않음).

## 명령어 상세

### set_line LINE:HASH

단일 줄 내용을 교체합니다.

```
set_line 5:a3
새로운 5번째 줄 내용
---
```

### replace_lines START:HH END:HH

줄 범위(포함)를 새 내용으로 교체합니다. 새 내용은 원본 범위보다 적거나 같거나 많은 줄을 가질 수 있습니다.
START ≤ END 이어야 합니다.

```
replace_lines 10:b2 15:c7
교체 줄 1
교체 줄 2
---
```

### insert_after LINE:HASH

앵커 줄 다음에 하나 이상의 줄을 삽입합니다.
동일 줄에 중복 `insert_after` 불가.

```
insert_after 20:d4
새 줄 1
새 줄 2
---
```

### delete_lines START:HH END:HH

줄 범위(포함)를 삭제합니다. 내용 줄 불필요.
START ≤ END 이어야 합니다.

```
delete_lines 25:e5 30:f6
---
```

## 성공 후 출력

성공 시 편집기 출력:
1. `OK` (첫 줄)
2. 영향받은 줄 + 전후 1줄 컨텍스트, **새** 줄 번호와 해시를 `LINE:HASH|content` 포맷으로

파일을 다시 읽지 않고 이 새 해시를 후속 편집에 바로 사용할 수 있습니다.

## 오류 출력

오류는 컨텍스트와 함께 stderr에 출력됩니다:

```
ERR anchor_mismatch 5:a3
3:xx|컨텍스트 줄
4:xx|컨텍스트 줄
5:yy|실제 내용 (해시 불일치)
6:xx|컨텍스트 줄
7:xx|컨텍스트 줄
```

## 후행 개행 보존

편집기는 원본 파일의 후행 개행 여부를 감지하여 편집 후에도 동일하게 유지합니다.
후행 개행이 없는 파일은 편집 후에도 없이 유지됩니다.

## 파일 권한

편집기는 임시 파일 작성 후 `mv`를 통한 원자적 교체로 원본 파일 권한을 보존합니다.
v0.3부터 임시 파일은 `/tmp`가 아닌 대상 파일과 같은 디렉토리에 생성되어, cross-device `mv` 실패를 방지합니다.
