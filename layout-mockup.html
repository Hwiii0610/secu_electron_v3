현재 레이아웃 문제점

① 상단바 72px 낭비

8개 탭이 아이콘+텍스트로 동일 높이 차지. 탐지모드(4개)와 액션(4개)이 시각적 구분 없음. 자주 전환하지 않는 기능(설정, 일괄처리)도 같은 레벨.

② 우측 사이드바 정적 정보

파일정보(이름,용량,해상도 등)는 작업 중 거의 참조 안 함. 상시 260px 차지하여 비디오 영역 축소.

③ 비디오 영역 압축

상72px + 우260px + 하150px 제외 시 영상이 실제 화면의 ~55%만 차지. 마스킹 정밀작업에 불리.

④ 추가/삭제 버튼 고립

파일 추가/삭제가 우측 하단에 별도 영역으로 존재. 파일목록과 물리적으로 떨어져 있음.

Option A — 좌측 아이콘 사이드바 (Figma/VS Code 스타일)
• 상단바 제거 → 좌측 52px 아이콘바로 모드 전환
• 우측 사이드바 제거 → 파일정보는 상단 컨텍스트바로 통합
• 영상 영역 최대화 (전체의 ~78%)
• 파일 아이콘: 좌측 사이드바 최상단 — 클릭 시 파일 목록 패널 토글
• 솔루션 로고: 컨텍스트바 중앙 텍스트
• 사명 로고 (SPHERE AX): 컨트롤바 우측 하단 — 기존 프레임모드/정보 영역 대체
• 프레임 정보: 컨텍스트바 우측 끝으로 이동 (▲▼ 5초 + F 5632)
• 탐지 팝업: 영상 영역 우측 상단 플로팅 오버레이 (진행률 바 포함)
  — 좌측 아이콘에도 pulse 인디케이터로 탐지 진행 상태 표시
