# SecuWatcher UX 크리틱 리포트

> 분석 대상: 파일 패널 오픈 상태 + 플로팅 컨트롤바 + 사이드바 레이아웃
> 분석 일자: 2026-03-06

---

## 1. First Impression (2초 테스트)

**시선이 가는 곳**: 파일 패널 내의 파란색 선택행 (#2 KakaoTalk...)이 가장 먼저 눈에 들어옴.
**긍정적**: 다크 테마가 영상 편집 도구에 적합하고, 전반적으로 깔끔한 레이아웃.
**문제점**: 비디오 영역이 완전히 가려져 있어 "영상 분석 앱"이라는 1차 목적이 즉시 전달되지 않음. 앱을 처음 본 사용자는 파일 관리 도구로 오인할 수 있음.

---

## 2. Usability 분석

### 2-1. 파일 패널이 비디오를 완전히 가림 (심각도: 높음)

- **현상**: `file-panel-container` width 310px이 비디오 영역 좌측을 완전히 덮고 있음
- **영향**: 파일 선택과 영상 확인을 동시에 할 수 없음 → 파일 클릭 → 패널 닫기 → 영상 확인 → 패널 열기 반복
- **개선안 A**: 패널 오픈 시 비디오 영역을 오른쪽으로 리사이즈 (오버레이 → 인라인 전환)
- **개선안 B**: 패널 너비를 축소하거나 반투명도를 높여 비디오가 일부 보이도록 처리
- **개선안 C**: 파일 전환 시 자동으로 패널 접기 (0.5초 딜레이 후)

### 2-2. 토글 탭(◁) 발견성 부족 (심각도: 중간)

- **현상**: 우측 중앙의 작은 화살표(24×48px)가 유일한 패널 토글 수단
- **영향**: 첫 사용자가 패널을 어떻게 닫는지 인지하기 어려움
- **개선안**: 토글 탭에 "파일" 텍스트 라벨을 세로로 추가하거나, 첫 실행 시 툴팁 가이드 표시

### 2-3. 파일 정보 영역의 "대기중..." 상태 (심각도: 중간)

- **현상**: 파일이 선택되어 있으나(#2) 파일 정보가 모두 "대기중..."으로 표시
- **영향**: 파일이 로드되고 있는지, 오류인지 사용자가 판단할 수 없음
- **개선안**: 로딩 스피너 + 진행률 표시, 또는 "파일 분석 중..." 등 명확한 상태 메시지

### 2-4. 네비게이션 버튼 기능 학습 곡선 (심각도: 낮음)

- **현상**: |◁, ◁|, |▷, ▷| 4개 버튼의 차이가 아이콘만으로는 즉시 구분 어려움
- **영향**: "전체 처음" vs "현재 구간 처음"의 차이를 아이콘으로 즉시 파악하기 어려움
- **개선안**: 호버 시 툴팁이 이미 있으므로 양호하나, 버튼 아래에 2글자 약어 라벨 추가 고려 (처음/구간/구간/끝)

### 2-5. "취소" 버튼의 모호한 기능 (심각도: 중간)

- **현상**: 우측 "취소" 버튼이 `handleCancel()` → `$emit('toggle-play')` 호출 (재생 토글)
- **영향**: "취소"라는 라벨에서 재생 토글 동작을 예상하기 어려움
- **개선안**: 기능에 맞는 라벨로 변경 (예: "정지" 또는 분할 작업 취소라면 "분할 취소")

## 3. Visual Hierarchy 분석

### 3-1. 파일 패널 내부 정보 밀도 불균형

- **상단 "파일 정보"**: 6개 필드가 모두 "대기중..."으로 공간만 차지
- **하단 "파일 목록"**: 6개 파일이 나열되어 있어 상대적으로 정보가 풍부
- **개선안**: 파일 정보 영역을 접을 수 있는 아코디언으로 만들어, 파일 목록 공간 확보

### 3-2. 플로팅 컨트롤바와 파일 패널의 시각적 분리 부족

- **현상**: 둘 다 반투명 다크 배경 + 유사한 border 스타일
- **영향**: 어디까지가 파일 패널이고 어디부터가 컨트롤바인지 경계가 모호
- **개선안**: 파일 패널에 좌측 accent border를 추가하거나, 배경 톤을 미세하게 차별화

### 3-3. 사이드바 아이콘 그룹핑

- **긍정적**: separator로 "탐지/마스킹" 그룹과 "내보내기/설정" 그룹이 구분됨
- **개선점**: "미리보기"가 구분선 아래에 독립적으로 존재 → 기능적으로 "탐지" 그룹과 더 가까우므로 그룹 위치 재검토

## 4. Consistency 분석

### 4-1. 폰트 사이즈 (이전 작업에서 13px 통일됨 — 양호)

| 영역 | 현재 | 상태 |
|------|------|------|
| 컨텍스트바 | 13px | ✅ 통일됨 |
| 플로팅 컨트롤바 액션 | 13px | ✅ 통일됨 |
| 사이드바 아이템 텍스트 | 11px | ⚠️ 더 작음 |
| 파일 패널 row-header/content | 10px | ⚠️ 더 작음 |
| 파일 패널 title | 11px | ⚠️ 더 작음 |

- **개선안**: 사이드바(11px)와 파일 패널(10-11px)은 보조 요소이므로 현재도 수용 가능하나, 최소 12px 이상으로 가독성 개선 여지 있음

### 4-2. 인터랙티브 요소 스타일 일관성

- 사이드바: 아이콘+텍스트 세로 배치, hover시 배경 변경
- 파일 패널 버튼: "추가"/"삭제" — border 기반 ghost 버튼
- 플로팅 컨트롤바: "분할"/"취소" — 동일 ghost 버튼 스타일
- **긍정적**: 버튼 스타일은 대체로 통일됨 (ghost 버튼 패턴)
- **개선점**: "삭제" 버튼만 빨간색 강조 → "분할"의 ✂ 아이콘 + 파란색과 대비되어 시각적 우선순위가 혼란스러울 수 있음

### 4-3. 색상 체계

- **파란색(#3A82C4)**: 주요 액션, 선택 상태, 플레이헤드
- **주황색(#E67E22)**: 파일명 강조, 활성 세그먼트 border, (이전) 토글 탭
- **빨간색(#EF4444)**: 삭제 버튼
- **개선점**: 주황색이 "파일명 강조"와 "활성 세그먼트"에 동시 사용되는데, 이 두 가지의 의미적 연관이 약함. 파일명은 브랜드 색상, 세그먼트는 상태 표시로 역할이 다르므로 구분 고려

## 5. 우선순위별 개선 액션

### 즉시 적용 가능 (코드 수정만으로 해결)

1. **"취소" 버튼 라벨/기능 명확화** — 재생 토글이면 "정지"로, 분할 취소면 분할 상태 초기화로
2. **파일 정보 "대기중..." 상태에 로딩 인디케이터 추가** — CSS 애니메이션 dot 등
3. **토글 탭에 텍스트 라벨 추가** — "파일" 세로 텍스트로 발견성 향상

### 단기 개선 (설계 조정 필요)

4. **파일 선택 시 자동 패널 접기** — selectFile 후 0.5초 딜레이 패널 토글
5. **파일 정보 영역 아코디언화** — 접기/펼치기로 공간 효율 개선
6. **파일 패널 row-header/row-content 폰트 12px로 상향**

### 중장기 개선 (아키텍처/레이아웃 변경)

7. **파일 패널 오버레이 → 인라인 전환 옵션** — 패널 오픈 시 비디오 영역 리사이즈
8. **Onboarding 가이드** — 첫 실행 시 주요 UI 요소 하이라이트 투어
9. **네비게이션 버튼 약어 라벨** — 공간이 허용되면 아이콘 아래 2글자 추가
